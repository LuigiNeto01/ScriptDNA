# Objetivo da análise

O sistema tem como objetivo gerar roteiros melhores usando todos os dados disponíveis do canal, vídeos, transcrições, métricas, histórico, insights e estilo.

A ideia esperada é um ciclo fechado:

1. importar/subir vídeos ou Shorts;
2. transcrever;
3. analisar beats narrativos;
4. salvar dados estruturados;
5. importar métricas de performance;
6. analisar performance;
7. gerar aprendizados/insights;
8. usar esses aprendizados na próxima geração de roteiro;
9. melhorar continuamente a geração.

Quero que você descubra se esse fluxo está realmente funcionando no código e quais dados estão sendo ignorados, mal usados ou poderiam melhorar a performance dos roteiros.
